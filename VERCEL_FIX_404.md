# Исправление ошибки 404 на Vercel

## Проблема
При развертывании на Vercel появляется ошибка:
```
404: NOT_FOUND
Code: NOT_FOUND
```

Согласно [официальной документации Vercel](https://vercel.com/docs/errors/NOT_FOUND), эта ошибка возникает когда запрашиваемый ресурс не найден.

## Решение

### Вариант 1: Использование Root Directory (РЕКОМЕНДУЕТСЯ)

1. Откройте ваш проект в [Vercel Dashboard](https://vercel.com)
2. Перейдите в **Settings** → **General**
3. Найдите секцию **Root Directory**
4. Нажмите **Edit** и введите: `front`
5. Сохраните изменения
6. Переразверните проект (Redeploy)

После этого Vercel будет использовать `front/vercel.json` и работать из директории `front/`.

### Вариант 2: Без Root Directory

Если вы не хотите использовать Root Directory, убедитесь что:

1. В корневом `vercel.json` правильно указаны пути:
   ```json
   {
     "buildCommand": "cd front && npm install && npm run build",
     "outputDirectory": "front/build"
   }
   ```

2. Переменные окружения установлены в Vercel Dashboard

3. Проект переразвернут после изменений

## Проверка

После применения исправлений, согласно [документации Vercel](https://vercel.com/docs/errors/NOT_FOUND):

1. **Проверьте URL развертывания:**
   - Убедитесь, что URL корректен и не содержит опечаток
   - Проверьте, что вы используете правильный домен

2. **Проверьте существование развертывания:**
   - Убедитесь, что deployment существует и не был удален
   - Проверьте статус deployment в Dashboard

3. **Проверьте логи сборки:**
   - Откройте Deployment → Build Logs
   - Убедитесь, что сборка проходит успешно
   - Проверьте, что команда `npm run build` выполняется без ошибок

4. **Проверьте структуру файлов:**
   - После сборки должна существовать директория `build` (или `front/build`)
   - В ней должен быть файл `index.html`
   - Проверьте, что `outputDirectory` в `vercel.json` указывает на правильную директорию

5. **Проверьте права доступа:**
   - Убедитесь, что у вас есть необходимые права для доступа к deployment

## Дополнительные шаги

Если проблема сохраняется, выполните следующие проверки:

1. **Проверьте логи развертывания:**
   - Откройте Deployment → Build Logs
   - Убедитесь, что команда `npm run build` выполняется успешно
   - Проверьте, что нет ошибок компиляции

2. **Проверьте структуру файлов:**
   - После сборки должна существовать директория `build` (или `front/build` в зависимости от Root Directory)
   - В ней должен быть файл `index.html`
   - Проверьте, что все статические файлы (JS, CSS) созданы

3. **Проверьте конфигурацию vercel.json:**
   - Убедитесь, что `outputDirectory` указывает на правильную директорию
   - Проверьте, что `rewrites` настроены для SPA
   - Если используется Root Directory, убедитесь, что используется правильный `vercel.json`

4. **Очистите кеш:**
   - В Vercel Dashboard → Settings → General
   - Нажмите "Clear Build Cache"
   - Переразверните проект

5. **Проверьте переменные окружения:**
   - Убедитесь, что `REACT_APP_API_URL` установлена
   - Проверьте, что она доступна для нужного окружения (Production/Preview/Development)
   - Убедитесь, что переменные не содержат опечаток

6. **Проверьте Framework Detection:**
   - В Settings → General проверьте, что Framework Preset установлен правильно
   - Для Create React App должен быть выбран "Create React App" или "Other"
